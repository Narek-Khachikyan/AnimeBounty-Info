# PRD: исправление проблем AnimeBounty-Info

## Цель

Стабилизировать React/Vite приложение AnimeBounty-Info для деплоя на Netlify: убрать падения на нестабильных ответах Jikan API, починить качество кода, безопасно собирать API-запросы и обновить уязвимые зависимости.

## Контекст деплоя

Проект деплоится на Netlify, не на GitHub Pages. Поэтому `public/_redirects` подходит для SPA-роутинга на Netlify. При этом в `package.json` остались GitHub Pages-настройки (`homepage`, `deploy`, `predeploy`, `gh-pages`), которые больше не отражают реальный способ деплоя и могут путать поддержку проекта.

## Ошибки и решения

### 1. Устаревшая конфигурация деплоя

**Проблема:** `package.json` содержит `homepage`, `deploy`, `predeploy` и зависимость `gh-pages`, хотя проект используется на Netlify.

**Риск:** разработчик может случайно ориентироваться на неправильную схему деплоя; лишняя зависимость увеличивает поверхность уязвимостей и maintenance.

**Решение:**

- удалить `homepage`, `deploy`, `predeploy`;
- удалить `gh-pages` из зависимостей;
- оставить `public/_redirects` для Netlify SPA fallback;
- при необходимости добавить `netlify.toml` с `publish = "dist"` и `command = "npm run build"`.

### 2. Страницы падают при ошибке Jikan API

**Проблема:** страницы проверяют в основном `isLoading` и happy path. Если API вернет ошибку или rate limit, дочерние компоненты получают `undefined` и вызывают `data.map()` или `data.slice()`.

**Риск:** пользователь видит белый экран вместо понятного состояния ошибки.

**Решение:**

- в `Anime.jsx`, `Manga.jsx`, `AnimeSearch.jsx`, `MangaSearch.jsx`, `FullAnime.jsx`, `FullManga.jsx` обрабатывать `isError`, `error`, `isFetching`;
- показывать отдельный error-state с кнопкой повторить запрос;
- дочерним компонентам передавать безопасные значения: `data ?? []`;
- внутри списков использовать `Array.isArray(data) ? data : []`.

### 3. Нет защиты от пустых вложенных полей API

**Проблема:** код напрямую читает поля вроде `title_english.length`, `trailer.url`, `trailer.images.maximum_image_url`, `genres.map`, `aired.string`, `images.webp.image_url`.

**Риск:** отдельные anime/manga карточки или страницы деталей могут падать, если Jikan вернет `null` или неполные данные.

**Решение:**

- использовать optional chaining: `item.title_english?.length`, `item.images?.webp?.image_url`;
- добавить fallback значения: `title_english || title || "No title"`;
- не рендерить trailer-блок, если нет `trailer?.url` или изображения;
- вынести форматирование API-ответов в helper-функции, чтобы компоненты получали уже нормализованные данные.

### 4. Поисковые URL собираются вручную

**Проблема:** в `apiSlice.js` query-string собирается строкой: `q=${query}`.

**Риск:** символы `&`, `?`, `#`, пробелы и другие спецсимволы ломают запрос или меняют параметры.

**Решение:**

- заменить ручную сборку на `URLSearchParams`;
- добавлять только непустые параметры;
- отдельно проверить поиск по строкам с пробелами и спецсимволами.

### 5. Infinite scroll использует устаревшее состояние

**Проблема:** `handleScroll` в рекомендациях замыкает начальный `isLoading`, а `useEffect` подписывается с пустым массивом зависимостей.

**Риск:** при быстром скролле могут запускаться несколько таймеров загрузки подряд; lint также показывает `react-hooks/exhaustive-deps` предупреждения.

**Решение:**

- заменить scroll listener на `IntersectionObserver`;
- либо завернуть `handleScroll` в `useCallback` и корректно указать зависимости;
- ограничить `visibleItems` длиной массива данных;
- вынести общий infinite-scroll код в переиспользуемый hook.

### 6. Lint сейчас не проходит

**Проблема:** `npm run lint` падает на 26 ошибках `react/prop-types` и 2 предупреждениях hooks.

**Риск:** CI/деплой легко пропустит реальные регрессии, потому базовая проверка уже красная.

**Решение:**

- добавить PropTypes во все компоненты, где они отсутствуют;
- исправить `react-hooks/exhaustive-deps`;
- рассмотреть переход на TypeScript, если проект планируется развивать дальше;
- после исправлений сделать `npm run lint` обязательной проверкой перед деплоем.

### 7. Уязвимые зависимости

**Проблема:** `npm audit` показывает 25 уязвимостей, включая critical в `@babel/traverse` и `swiper`, high в `react-router`, `@remix-run/router`, `rollup`, `cross-spawn` и других пакетах.

**Риск:** устаревший frontend-стек несет security и maintenance риски; часть проблем исправляется обновлением lockfile, часть требует major upgrade.

**Решение:**

- выполнить `npm audit fix` и проверить сборку;
- отдельно обновить `swiper` до актуальной major-версии и адаптировать импорты, если API изменился;
- обновить `vite`, `@vitejs/plugin-react`, `react-router-dom`, `eslint` и сопутствующие dev-зависимости;
- после обновлений прогнать `npm run build`, `npm run lint`, ручную проверку основных маршрутов.

## Acceptance Criteria

- `npm run build` успешно проходит.
- `npm run lint` успешно проходит без ошибок и предупреждений.
- `npm audit --audit-level=high` не показывает high/critical уязвимостей.
- Netlify деплой открывает главную страницу и вложенные маршруты после refresh.
- При ошибке Jikan API приложение показывает понятное состояние ошибки, а не белый экран.
- Поиск корректно работает со строками, содержащими пробелы и спецсимволы.
- Страницы деталей не падают при отсутствии trailer, title_english, genres, images или aired.

## Приоритет работ

1. Исправить error-state и null-guards для Jikan API.
2. Починить lint и hook dependencies.
3. Переписать сборку query-параметров через `URLSearchParams`.
4. Обновить уязвимые зависимости.
5. Очистить GitHub Pages-наследие и зафиксировать Netlify-конфигурацию.
